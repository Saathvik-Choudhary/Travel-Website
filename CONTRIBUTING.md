# Contributing to Biking Adventure Website

Thanks for taking the time to look at **Biking Adventure Website**! 🙌 This is an actively evolving
project and **feedback is genuinely wanted** — whether you write code or not.

## The easiest way to help: tell me what you think

You don't need to open a pull request to contribute. The most valuable thing right now
is **your opinion**:

- 💬 **[Start a Discussion](https://github.com/Saathvik-Choudhary/Travel-Website/discussions)** — ideas, questions, "I wish it did X", or just first impressions.
- 💡 **[Open a Feedback issue](https://github.com/Saathvik-Choudhary/Travel-Website/issues/new?template=feedback.md)** — a structured way to share a suggestion.
- 🐞 **[Report a bug](https://github.com/Saathvik-Choudhary/Travel-Website/issues/new?template=bug_report.md)** — found something broken? Let me know.
- ⭐ **Star the repo** if you find it interesting — it helps others discover it.

## Contributing code

1. Fork the repo and create a branch: `git checkout -b feature/your-idea`
2. Make your change. Keep it focused and match the existing style.
3. Run the project locally and confirm it still works (see the README).
4. Commit with a clear message and open a pull request describing **what** and **why**.

I review PRs as a solo maintainer, so small, well-scoped changes get merged fastest.

## Ground rules

- Be kind and constructive — see the spirit of the [Contributor Covenant](https://www.contributor-covenant.org/).
- No personal data, secrets, or API keys in issues, PRs, or commits.
- Not sure if something is welcome? Open a Discussion and ask — there are no dumb questions.

Thank you for helping make Biking Adventure Website better! 💙
